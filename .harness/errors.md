## 已登记错误

### ERR-001: 校准页 API 传参不匹配

**时间**: 2026-06-04
**任务**: 绩效系统代码审查
**错误类型**: 猜测
**错误详情**: 前端校准弹窗提交的数据结构是 `{ user_id, perf_score, value_grade, reason }`，但后端 `CalibrateItem` 期望三维度分开 `{ user_id, perf_score, value_belief_grade, value_team_grade, value_growth_grade, reason }`。
**根因分析**: 前后端契约没有对照表，开发者猜测价值观应该和业绩一样用一个字段
**修复方案**: 前端改为提交三个独立字段
**对应规则**: RULE-012 规则一（不懂就问）+ RULE-001（字段一致性）
**预防措施**: 所有涉及前后端接口的任务，Plan 阶段必须输出字段对照表

### ERR-002: 首页价值观显示为 "-"

**时间**: 2026-06-04
**任务**: 绩效系统代码审查
**错误类型**: 猜测
**错误详情**: 后端 `CycleParticipant` 模型存的是三个独立字段 `final_value_belief`、`final_value_team`、`final_value_growth`，但前端代码用了不存在的 `final_value_grade`
**根因分析**: 开发者猜测价值观和业绩一样有单一等级字段
**修复方案**: 前端改为读取三个独立字段并展示
**对应规则**: RULE-012 规则一（不懂就问）
**预防措施**: 涉及模型字段的修改，必须同时检查前后端所有引用该字段的地方

### ERR-003: 互评提交 schema 与模型不匹配

**时间**: 2026-06-08
**任务**: 绩效系统功能差距评估
**错误类型**: 未标明不确定 / 遗留改造
**错误详情**: 后端数据库模型 `PeerEvaluation` 已按价值观三维度改造（`value_belief_grade`/`value_team_grade`/`value_growth_grade`），但 API 的 `PeerSubmit` schema 和提交逻辑仍使用单维度 `value_grade`/`value_example`，导致互评人提交后三维度字段始终为空。
**根因分析**: 价值观三维度改造过程中，互评模块未同步升级 schema 和提交逻辑
**修复方案**: 后端 `PeerSubmit` schema 改为三维度；前端 `PeerTasks.tsx` 同步改为三维度表单
**对应规则**: RULE-001（字段一致性）
**预防措施**: 数据模型改造必须同步检查所有相关的 schema、API、前端表单和展示逻辑

### ERR-004: 导出/历史查询引用已删除字段

**时间**: 2026-06-08
**任务**: 绩效系统功能差距评估
**错误类型**: 未标明不确定 / 遗留改造
**错误详情**: `export.py` 和 `history.py` 引用模型已删除的 `final_value_grade` 字段，而模型已拆分为 `final_value_belief`/`final_value_team`/`final_value_growth`，导致接口 500
**根因分析**: 模型字段删除后，没有全局搜索所有引用位置
**修复方案**: 改为读取三个独立字段，导出 Excel 增加三列/历史查询返回三维对象
**对应规则**: RULE-001（字段一致性）
**预防措施**: 删除/重命名模型字段前，全局搜索前后端所有引用并同步修改
