## 1. 项目概述

PMS 绩效管理系统是公司自研的人力资源绩效管理应用，面向员工、直属上级、HRBP/HR 管理员及部门 Leader 等多角色，覆盖目标制定、绩效自评、上级评估、互评、绩效校准、周期管理、绩效看板等核心流程。本次重设计以“让绩效信息清晰易读、让管理工作高效可见”为目标，在保持业务逻辑不变的前提下，重塑信息层级、重构导航体系、统一视觉语言，并显著提升表格、表单、数据可视化的使用效率。

设计输出采用 **HTML 高保真原型 + 设计令牌文档** 的形式，共 6 个核心页面。所有 HTML 页面均引用同目录下的 `styles.css`，图标采用内联 SVG，无任何外部 CDN、字体或图片依赖，可直接在浏览器中双击打开预览。本交付包同时包含《KIMI-EXEC-PLAN.md》，供后续开发侧 Kimi 按图施工。

## 2. 设计系统选择说明

### 2.1 结构参考：Linear

- **导航模式**：左侧边栏（240px 展开 / 72px 收起）+ 顶部全局栏（56px）+ 内容区，与飞书 People 一致，适合多角色工作台。
- **信息密度**：在干净界面中展示大量任务状态、优先级、负责人，与绩效管理系统中的待办、评估、校准列表高度契合。
- **组件结构**：状态徽章、表格、分面筛选、Timeline、Progress 等组件与 PMS 核心需求重叠。

### 2.2 视觉语言：飞书绩效 / 飞书 People

- **色彩**：以 `#3370FF` 为主色，背景 `#F5F6F7`，卡片白底，使用低饱和语义色（成功/警告/危险/信息）。
- **字体**：系统字体栈，正文 14px，关键数字使用 24-28px，保证跨平台一致性。
- **圆角与阴影**：默认 8px 圆角，1px 边框，使用极轻阴影（`0 1px 2px rgba(31,35,41,0.04)`）。
- **交互**：悬停、选中、聚焦状态明确，按钮/表单统一，避免过度装饰。

### 2.3 选择结论

最终方案以 **Linear 作为结构参考底 + 飞书绩效/People 视觉语言为主线**。Linear 提供了信息架构与组件组织方式，飞书提供了“轻商务、温暖、克制”的视觉气质，两者结合形成“现代、专业、高效”的 PMS 设计语言，并可直接映射到 Ant Design 5 组件库。

## 4. 响应式设计说明

### 4.1 设计目标

本次在 v1.0 视觉规范基础上，针对桌面端与移动端浏览器补全响应式体验：

- **桌面端**：保持高信息密度，左侧边栏常驻，多列网格布局，复杂操作 inline 可见。
- **移动端**：抽屉侧栏 + 单列布局，表格卡片化，图表简化，长表单分步，底部固定操作栏方便单手操作。
- **中间态**：平板/小桌面在"侧栏可收起"与"多列 → 单列"之间平滑过渡。

### 4.2 断点策略

| 断点 | 宽度范围 | 布局策略 |
|------|----------|----------|
| `xl` | `>= 1280px` | 完整左侧边栏（240px）+ 内容区居中，最大宽度 1440px |
| `lg` | `1024px - 1279px` | 侧栏可展开/收起，多列网格保持 |
| `md` | `768px - 1023px` | 侧栏隐藏为抽屉，网格压缩为 2 列，底部操作栏出现 |
| `sm` | `640px - 767px` | 严格单列，表格卡片化，KPI 横向滑动 |
| `xs` | `< 640px` | 紧凑单列，触控目标 44-48px，字号微调 |

> 实现说明：`styles.css` 中使用 `max-width: 1023px`、`max-width: 767px`、`max-width: 639px` 与 `min-width: 1280px` 四个媒体查询，分别对应 1024px / 768px / 640px / 1280px 断点；与 `README.md` 和 `RESPONSIVE-SUMMARY.md` 保持一致。

## 5. 核心页面说明

### 5.1 首页 / 个人工作台（home.html）

- **设计重点**：角色切换为“员工”，顶部欢迎卡片 + 快捷入口，降低认知负荷。
- **信息层级**：KPI 迷你卡片 → 我的待办 → 试用期/绩效概览 → 我参与的周期。
- **交互**：待办任务支持“去处理”直达自评页；任务项悬停提升；卡片与表格统一。
- **数据呈现**：用 Timeline 展示周期里程碑，用 Tag 区分状态。

### 5.2 HR 绩效看板（dashboard.html）

- **设计重点**：面向 HRBP/管理者，用数据驱动决策。
- **信息层级**：KPI 行 → 部门进度对比 + 绩效等级分布 → 近周期趋势 → 部门概览表。
- **数据可视化**：内联 SVG 折线图、CSS 环形图、双进度条对比，均使用图表色板。
- **状态表达**：表格中使用 Tag 组合展示完成率与风险等级。

### 5.3 HR 管理台 / 周期管理（hr-console.html）

- **设计重点**：周期 CRUD + 参与人管理，强调批量操作与效率。
- **信息层级**：页头操作 → 周期 Tabs + 筛选 → 周期表格 → 批量导入区 → 选中周期参与人详情。
- **交互**：表格行内操作使用图标按钮分组；表格支持复选框；提供分页与导入导出入口。
- **状态区分**：周期状态使用 Primary/Warning/Success/Default Tag 区分。

### 5.4 员工自评页（self-eval.html）

- **设计重点**：长表单工作流，使用步骤条明确当前进度。
- **信息层级**：步骤条 → 绩效目标只读 → 自评内容（业绩/关键成果）→ 价值观评估 → 互评人 → 上级评估结果。
- **交互**：5 分制评分按钮，选中态高亮；文本域预填示例；长表单提供底部/顶部双提交入口思路。
- **可视化**：目标表格使用权重徽章、状态 Tag。

### 5.5 上级评估页（leader-eval.html）

- **设计重点**：左右分栏（下属列表 + 详情面板），提升评估效率。
- **信息层级**：下属列表（状态筛选）→ 员工详情 → 绩效目标 → 员工自评摘要 → 互评汇总 → 上级评估表单。
- **交互**：下属列表支持搜索；选中项高亮；详情面板底部固定操作栏。
- **数据呈现**：互评汇总使用卡片化评论；评分使用下拉选择；绩效等级建议使用评分按钮。

### 5.6 绩效校准页（calibration.html）

- **设计重点**：面向部门 Leader/HRBP，校准评分分布与异常。
- **信息层级**：校准状态栏 → KPI 行 → 实际 vs 目标分布对比 + 校准进度时间线 → 人员校准矩阵 → 审批操作。
- **数据可视化**：双向条形图对比实际/目标分布；矩阵中使用迷你条形图展示评分。
- **交互**：异常行高亮；输入框可调整校准后评分；底部审批操作明确。

4. **少量死代码**：`hr-console.html` 中存在未使用的 `.status-tag` 等样式，可在开发时清理。

## 9. 使用方式

1. 在 Finder 中打开 `/Users/trentshen/Documents/Kimi code - 工作区/hr/design-prototype/`。
2. 双击任意 `.html` 文件即可在浏览器中预览，无需启动服务器或联网。
3. 页面间跳转使用左侧边栏导航，相对路径均正确。
4. 开发侧以 `KIMI-EXEC-PLAN.md` 为主要施工依据，`styles.css` 中的 CSS 变量可直接复制到项目全局样式。
