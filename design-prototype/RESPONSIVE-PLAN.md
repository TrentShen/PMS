# PMS 响应式设计优化方案

> 基于现有原型（v1.0），针对桌面端与移动端的使用场景，给出响应式优化建议。
> 本方案仅作为设计参考，未修改现有项目文件。

---

## 一、当前响应式实现总结

现有原型已具备基础响应式能力：

| 断点 | 当前实现 |
|------|---------|
| **> 1024px** | 完整左侧边栏（240px）+ 顶部全局栏（56px）+ 多列内容 |
| **640px - 1024px** | 侧栏隐藏为抽屉，顶部显示汉堡菜单，内容区保持多列但适当压缩 |
| **< 640px** | 单列布局，KPI 卡片和网格自动堆叠，页面 padding 缩小 |

已覆盖：侧栏折叠、汉堡菜单、网格自动换行、内容区 padding 调整。

---

## 二、桌面端设计建议（> 1024px）

### 2.1 布局策略

- **左侧边栏固定 240px**：按角色分组显示导航（我的 / 评估 / 管理），当前角色高亮。
- **顶部全局栏 56px**：左侧放 Logo/标题、搜索框；右侧放通知、角色切换、头像。
- **内容区最大宽度 1440px**，居中，左右留白自动适应大屏幕。
- **多列卡片布局**：首页用 2-3 列网格，看板用 KPI 卡 + 图表组合。

### 2.2 信息密度

- 桌面端可以适当提高信息密度：表格行高 48px，KPI 卡 4 列，表单左右分栏。
- 关键操作按钮常驻可见，不需要额外展开。
- 长表单使用左侧步骤条 + 右侧内容区的左右分栏（类似飞书 People 设置页）。

### 2.3 交互优化

- 表格列可自定义显示/隐藏，减少横向滚动。
- 图表支持悬停（hover）显示 tooltip。
- 模态框居中显示，宽度根据内容自适应（小/中/大三种尺寸）。
- 批量操作工具栏常驻在表格上方。

### 2.4 多屏利用

- 绩效看板：左侧筛选器 + 右侧 KPI/图表，可折叠筛选器。
- 上级评估：左侧下属列表 + 右侧详情面板，可独立滚动。
- 校准页：顶部分布图 + 下方人员矩阵，矩阵支持固定表头横向滚动。

## 三、移动端设计建议（< 640px）

### 3.1 布局策略

- **顶部栏固定**：高度 56px，左侧汉堡菜单，中间标题，右侧通知/头像。
- **侧边栏变为抽屉**：从左侧滑出，覆盖主内容，带遮罩层，点击外部关闭。
- **内容区单列**：所有卡片、表格、表单一列堆叠。
- **底部操作栏**：长表单（自评、评估）在底部固定保存/提交按钮，方便单手操作。

### 3.2 导航简化

- 侧边栏菜单分组折叠，默认只展开当前角色相关分组。
- 首页增加快捷入口图标区（类似飞书 App 首页的九宫格）。
- 减少层级，优先展示“待办”和“下一步行动”。

### 3.3 表格处理

- 管理型表格（HR 管理台、校准矩阵）在移动端改为**卡片列表**。
- 每张卡片显示最关键 3-4 个字段，操作按钮放在卡片底部。
- 支持横向滚动作为兜底方案，但优先卡片化。

### 3.4 表单处理

- 所有表单单列显示，标签在上、输入框在下。
- 评分步骤条改为垂直或下拉选择，减少横向空间占用。
- 长表单分步骤展示，底部显示进度指示（第 2/4 步）。
- 日期选择器、下拉选择使用原生组件或底部弹窗。

### 3.5 图表处理

- 趋势图、分布图在移动端改为横向可滚动，或简化显示最近 3-4 个数据点。
- 环形图保留，但图例放在图表下方而不是右侧。
- 关键指标卡片改为横向滑动（类似飞书 dashboard 的指标卡轮播）。

### 3.6 操作反馈

- Toast 提示从顶部改为底部弹出，避免遮挡顶部栏。
- 确认弹窗使用底部操作表（Action Sheet）替代居中模态框。
- 加载占位使用骨架屏，避免布局跳动。

## 四、各页面响应式策略

| 页面 | 桌面端 | 移动端 |
|------|--------|--------|
| **首页** | 2-3 列网格：欢迎卡 + 待办 + 试用期 + 周期列表 | 单列堆叠；待办任务优先展示；快捷入口改为图标九宫格；周期列表卡片化 |
| **绩效看板** | 左侧筛选器 + 右侧 KPI/图表/表格 | 顶部筛选器；KPI 卡横向滑动；图表简化；部门表格卡片化 |
| **HR 管理台** | 周期表格 + 参与人 Tab；批量操作工具栏 | 周期卡片列表；参与人折叠在卡片内；操作按钮放入底部 Action Sheet |
| **员工自评** | 左右分栏：左侧步骤条 + 右侧表单 | 单列步骤条（顶部进度）；底部固定操作栏；分步骤填写 |
| **上级评估** | 左侧下属列表 + 右侧详情面板 | 下属列表页 → 详情页；详情页分区块折叠；评分用底部弹窗 |
| **绩效校准** | 顶部分布图 + 人员矩阵表格 | 分布图简化；人员矩阵卡片化；审批按钮固定在底部 |

## 五、给 Kimi 的可执行改造项（响应式）

### 5.1 全局改造

1. **断点补充**
   - 当前断点：`1024px`、`640px`。
   - 建议增加：`768px`（平板竖屏）作为中间态，管理表格从 2 列切换到 1 列。

2. **容器类扩展**
   - 新增 `.container-responsive`：桌面 `max-width: 1440px`，移动端 `padding: 16px`。
   - 新增 `.mobile-only` / `.desktop-only` 工具类，用于控制元素显隐。

3. **抽屉侧栏完善**
   - 侧栏打开时添加 `body` 级遮罩，点击遮罩关闭。
   - 侧栏打开时锁定 `body` 滚动。
   - 侧栏滑入动画：`transform 0.25s ease`。

4. **字体响应式**
   - 桌面标题：`24px` / `20px` / `16px`。
   - 移动标题：`20px` / `18px` / `15px`。
   - 正文保持 `14px`，最小不小于 `13px`。

### 5.2 组件改造

| 组件 | 桌面端 | 移动端 |
|------|--------|--------|
| **Table** | 完整列，行高 48px，操作列图标 grouping | 改为 `MobileCardList` 组件；关键字段 3-4 个；操作按钮底部排列 |
| **KPI Card** | 4 列或 2 列网格 | 横向滑动或 2 列网格 |
| **Form** | 两列布局，标签右对齐 | 单列，标签上对齐，底部固定操作栏 |
| **Chart** | 完整趋势图/分布图，图例在右侧 | 横向滚动或简化数据点，图例在下方 |
| **Modal** | 居中，宽度 480/640/800px | 底部全宽抽屉（Drawer）或 Action Sheet |
| **Filter** | 顶部水平筛选器 | 顶部一个“筛选”按钮，点击展开抽屉 |

### 5.3 页面改造

1. **首页（Home）**
   - 桌面：欢迎卡 + 2 列网格（待办 + 试用期）+ 周期表格。
   - 移动：欢迎卡简化；待办列表优先；试用期进度条；周期列表改为卡片，每张卡片显示周期名、状态、我的结果。

2. **绩效看板（Dashboard）**
   - 桌面：KPI 4 列 + 趋势图 + 分布图 + 部门表格。
   - 移动：KPI 横向滑动（4 张卡）+ 趋势图（最近 3 期）+ 分布图简化 + 部门卡片列表。

3. **HR 管理台（HrConsole）**
   - 桌面：周期表格 + 参与人 Tab 表格。
   - 移动：周期卡片列表，点击展开参与人列表；批量操作隐藏，单条操作使用 Action Sheet。

4. **员工自评（SelfEval）**
   - 桌面：左侧步骤条 + 右侧表单。
   - 移动：顶部步骤进度条（只显示当前步骤名）+ 分步表单 + 底部固定“上一步/下一步/保存”。

5. **上级评估（LeaderEval）**
   - 桌面：左列表 + 右详情。
   - 移动：列表页 → 详情页；详情页分区块（目标、自评、互评、评分），使用折叠面板。

6. **绩效校准（Calibration）**
   - 桌面：顶部分布图 + 人员矩阵表格。
   - 移动：分布图简化（柱状图）+ 人员卡片列表；审批按钮固定在底部。

## 六、验收标准

### 6.1 桌面端

- [ ] 在 1920×1080 下，内容区居中，最大宽度 1440px，不拉伸变形。
- [ ] 在 1366×768 下，侧边栏完整显示，表格无横向滚动。
- [ ] 在 1024×768 下，侧边栏可收起，关键操作仍可见。

### 6.2 移动端

- [ ] 在 375×812（iPhone X）下，所有页面可正常浏览，无横向滚动。
- [ ] 在 414×896（iPhone Max）下，KPI 卡和图表可横向滑动或自动适配。
- [ ] 在 iPad 竖屏（768×1024）下，侧边栏可收起，表格可横向滚动或卡片化。
- [ ] 所有操作按钮在拇指可触范围内（底部 60-80px 区域优先）。

---

## 七、注意事项

1. **移动端不是桌面端缩小版**：需要重新组织信息优先级，而不是简单缩放。
2. **表格优先卡片化**：移动端表格体验差，应优先使用卡片列表。
3. **长表单分步**：避免一屏内塞入过多字段，增加用户放弃率。
4. **触控目标最小 44×44px**：按钮、链接、图标点击区域要足够大。
5. **性能考虑**：移动端减少图表数据点，避免复杂 SVG 渲染卡顿。
