## 二、冲刺目标

### 2.1 主目标：月底绩效启动会可用

在 6 月 30 日前，让系统支持一个完整绩效周期的 **happy path**：

```
HR 创建周期 → 员工录入/导入目标 → 员工自评 → 互评 → 上级评估 →
Leader 校准 → HR/CEO 审批 → 1:1 面谈 → 结果发布 → 员工确认/申诉
```

### 2.2 兜底目标：试用期管理可用

若主目标因企微配置/数据准备等原因无法达成，则确保系统至少能跑通：

```
试用期员工目标设定 → 导师/上级月度评价 → HR 查看进度 → 转正前综合评议
```

试用期管理需要的功能子集：
- 组织架构与人员管理（可不依赖企微，手动导入）
- 目标设定与审批
- 上级评估（业绩 + 价值观三维度）
- 历史绩效查看
- 结果导出 Excel

## 七、关键决策点

### 决策 1：月底是否必须跑完整绩效周期？
- **推荐**：能跑完整流程最好，但若外部依赖阻塞，优先保证**目标设定 + 自评 + 上级评估 + 结果发布**的最小闭环。

### 决策 2：互评环节是否启用？
- **推荐**：若时间紧，可先关闭互评环节，下月迭代开启。当前互评代码已较完整，开启成本不高。

### 决策 3：试用期管理是否作为独立模块？
- **推荐**：不单独开发模块，复用现有「目标 + 评估 + 导出」能力，只调整流程命名和周期模板。
