# 绩效管理系统（Performance Management System）—— 产品需求文档（PRD）v1.2（完整确认版）

> **文档状态**：核心规则已全部确认，可进入设计/开发阶段  
> **适用对象**：100人以内规模企业  
> **参考产品**：飞书People（绩效模块）  
> **集成平台**：企业微信（核心入口）

## 一、项目背景与目标

### 1.1 背景
- 公司目前绩效管理以 **Excel 线下操作** 为主，目标设定、评分汇总、结果校准均依赖人工，效率低、易出错、难追溯。
- 随着组织发展，需要一套线上化、流程化的绩效系统，将「目标设定 → 评估执行 → 结果校准 → 反馈沟通 → 人才盘点」全链路搬到线上。

### 1.2 目标
| 目标 | 描述 |
|------|------|
| **流程线上化** | 自评、互评、上级评估、校准、反馈全流程线上闭环 |
| **权限精细化** | 严格数据隔离：个人仅见自己，上级见下属，部门Leader见部门范围 |
| **效率提升** | 自动消息提醒、自动汇总算分、Excel一键导入历史目标，降低HR手工操作 |
| **决策支持** | 历史绩效可追溯，支持人才盘点（绩效-潜力矩阵等） |
| **体验无缝化** | 嵌入企业微信，员工无需额外安装App |

## 二、用户角色与权限模型

### 2.1 角色定义

| 角色 | 定义 | 典型人员 |
|------|------|----------|
| **超级管理员** | 系统配置、角色分配、全局数据可见 | IT/HR负责人 |
| **HRBP/绩效管理员** | 创建绩效项目、配置流程、监控进度、导出报表、发起催办 | HR |
| **部门Leader（校准人）** | 查看部门成员绩效、参与校准会议、调整绩效结果 | 部门负责人 |
| **直属上级** | 查看下属信息、进行上级评估、查看下属自评与互评结果 | 各层级主管 |
| **普通员工** | 查看自己的绩效目标、提交自评、进行互评、查看自己的最终绩效结果 | 全员 |

### 2.2 数据权限规则（核心原则：最小可见原则）

```
权限层级（由高到低）：
超级管理员 > HRBP/绩效管理员 > 部门Leader（校准人） > 直属上级 > 普通员工

【规则说明】
1. 普通员工：仅能查看自己的绩效目标、自评内容、自己的互评任务、自己的最终绩效结果。
2. 直属上级：可查看自己所有直接下属的上述全部信息，以及下属收到的互评结果。
3. 部门Leader：可查看本部门（含所有子部门）所有成员的绩效数据，参与校准。
4. HRBP：按配置的管辖范围查看数据，通常可跨部门查看。
5. 超级管理员：全局可见，主要用于系统维护与问题排查。
```

## 三、功能模块详述

### 3.1 组织架构与人员管理

#### 3.1.1 组织架构同步
- **来源**：对接企业微信组织架构（部门树、汇报关系），作为系统内权限判断的基础。
- **同步方式**：定时自动同步（每日凌晨）+ 手动同步按钮。
- **关键字段**：员工ID、姓名、部门、直属上级、职级/序列、入职日期、企业微信UserID。
- **状态**：✅ 已确认——企业微信组织架构和汇报关系已维护完整，无矩阵式/虚线汇报。

#### 3.1.2 汇报关系与权限继承
- 绩效系统的「直属上级」严格跟随企业微信的汇报关系。
- 部门Leader的判定：取企业微信中该部门的「负责人」字段，若未设置，可由HRBP在系统中手动指定。

---

### 3.2 绩效项目管理

#### 3.2.1 项目创建
- HRBP/绩效管理员可创建绩效项目，核心配置项：

| 配置项 | 说明 |
|--------|------|
| **项目名称** | 如「2025年下半年度绩效考核」 |
| **考核周期** | 开始日期、结束日期 |
| **考核对象** | 可选择全员参与，或排除指定人员/职级（如排除高管） |
| **考核模式** | 自评 + 互评 + 上级评估 + 校准 + 反馈（可开关各环节） |
| **互评规则** | 员工自选邀请（≤5人）→ Leader审核确认 → 发起互评 |
| **评估维度** | 业绩（1-5分，0.25分段）+ 价值观（甲/乙/丙） |
| **评分规则** | 业绩1-5分（0.25分段）；价值观甲/乙/丙 |
| **强制分布规则** | 3-6-1（A≤30%、B≈60%、C≥10%） |
| **结果可见性** | 最终绩效结果何时对员工可见（校准完成后/反馈完成后） |

#### 3.2.2 考核对象配置
- **默认选项**：全员参与。
- **排除模式**：可按人员、按职级、按部门排除特定人群。
- **入职时间门槛**：支持按入职日期过滤（如入职不晚于考核周期首月1日）。系统可配置该门槛，不满足条件的员工自动排除或标记为"参考试用期目标"。
- **典型场景**：高管通常由CEO/董事会评估，不参与常规互评流程，可通过排除模式将其移出。

#### 3.2.3 时间线与自动提醒
- 每个绩效项目可配置各环节的时间节点，示例：

```
【2025年下半年度绩效考核】时间线
├─ 目标确认：12-01 ~ 12-10
├─ 自评环节：01-06 ~ 01-13
├─ 互评名单确认：01-14 ~ 01-15（Leader审核员工邀请的互评人）
├─ 互评环节：01-16 ~ 01-20
├─ 上级评估：01-21 ~ 01-27
├─ 绩效校准（部门级）：01-28 ~ 02-05
├─ 绩效校准（公司级审批）：02-06 ~ 02-10（HR负责人 → CEO串行审批）
├─ 绩效反馈：02-11 ~ 02-18
└─ 结果公布：02-19
```

- **自动消息提醒**：
  - 环节开始前1天：提醒参与者即将开始。
  - 环节截止前3天、1天：提醒尚未完成的人员尽快提交。
  - 环节截止当天：提醒上级/HRBP有哪些人逾期未完成。
  - **消息推送渠道**：企业微信应用消息（见 3.7 节）。

- **催办功能**：HRBP/部门Leader/直属上级可手动对未完成人员发起催办。

### 3.3 绩效目标设定（支持Excel导入）

#### 3.3.1 首次上线：Excel批量导入（一次性功能，简化设计）
- **适用场景**：系统首次上线，历史绩效目标均以Excel形式存在。**本功能为一次性过渡方案，后续迭代转为线上化目标设定。**
- **简化设计原则**：
  - 不做复杂的版本管理、在线编辑、审批流。
  - 仅支持单次导入，导入后员工可查看，无需在系统内做在线确认。
  - 错误处理：校验失败直接返回错误文件，不支持部分导入。
- **操作流程**：
  1. HRBP下载系统提供的「绩效目标导入模板」。
  2. 按模板格式填写：员工ID、姓名、目标项、目标描述、衡量标准、权重、截止日期。
  3. 上传Excel，系统进行格式校验（必填项、权重总和是否为100%、员工ID是否存在等）。
  4. 校验通过后一键导入；校验失败则返回错误行提示，支持下载错误文件修正后重新上传。
- **模板字段**：

| 字段 | 必填 | 说明 |
|------|:--:|------|
| 员工ID | ✅ | 与企业微信一致 |
| 姓名 | ✅ | 用于人工核对 |
| 目标类别 | ❌ | 如：业绩目标/个人发展目标 |
| 目标项 | ✅ | 目标标题 |
| 目标描述 | ✅ | 具体描述 |
| 衡量标准 | ✅ | 如何算达成 |
| 权重(%) | ✅ | 单项目标权重，同一人的权重总和应为100 |
| 目标周期 | ✅ | 如：2025H2 |
| 截止日期 | ❌ | 目标预计完成时间 |

#### 3.3.2 目标设定要求（基于实操指南）
- **SMART原则**：具体（Specific）、可衡量（Measurable）、可达成（Achievable）、相关联（Relevant）、有时限（Time-bound）。
- **数量**：考核项以 **3-5项** 为最佳。
- **评分标准预设**：设定目标时需明确4分（100%符合预期）和5分（远超预期/优秀）的对应标准。
- **关键成果关联**：关键成果必须与目标设定有强关联性；"做了什么"很重要，"做成了什么更关键"。

#### 3.3.3 线上化目标设定（V1.2 迭代）
- 支持员工在系统内直接填写目标，上级在线审批确认。
- 目标调整：考核周期内，经上级审批后可调整目标内容或权重，保留修改记录。

### 3.4 绩效评估流程

#### 3.4.1 评估维度与评分标准

##### 维度1：业绩评估（1-5分，0.25分段）

| 业绩等级 | 分数区间 | 定义 |
|:--------:|:--------:|------|
| **优秀** | >4.5 | 显著超过预期绩效目标或职责要求，并在目标达成过程中价值观行为堪称典范，对团队其他成员有积极的影响作用。 |
| **部分超出预期** | 4.0 - 4.5（含） | 部分超过预期绩效目标或职责要求，并在目标达成过程中价值观行为堪称典范，对团队其他成员有积极的影响作用。 |
| **符合预期** | 3.5 - 4.0（含） | 达到预期职责目标或绩效要求，在目标或职责要求所涉及的主要方面取得较突出的成绩，并在过程中一贯保持良好的价值观，能够对团队产生一定的影响。 |
| **部分不符合预期** | 3.0 - 3.5（含） | 达到预期绩效目标或岗位职责的最低要求，在工作绩效、专业技能方面仍有提高的潜力和必要。 |
| **不符合预期** | ≤3.0 | 不符合本岗位的岗位要求，无法完成预期目标，或在价值观方面不能满足公司的要求，在团队内部带来不良的影响，予以淘汰。 |

- **评分规则**：以0.25为一个分数段，最终分数结果需为0.25倍数。
- **关键成果要求**：自评及上级评估中，均需说明关键成果，并确保关键成果与目标设定有强关联性。

##### 维度2：价值观评估（甲/乙/丙）

| 等级 | 说明 |
|:--:|------|
| **甲** | 价值观行为堪称典范，需在其中提供具体事例 |
| **乙** | — |
| **丙** | — |

- **特殊规则**：价值观评级为**乙**的员工，将失去接下来的所有晋升及调薪资格。
- **联动规则**：价值观评级可能影响业绩等级的最终评定（业绩等级定义中已内嵌价值观要求）。

##### 强制分布映射（3-6-1）

| 强制分布档位 | 业绩等级范围 | 占比要求 |
|:----------:|:-----------|:-------:|
| **A（前30%）** | 优秀 + 部分超出预期 | ≤30% |
| **B（中间60%）** | 符合预期 | ≈60% |
| **C（后10%）** | 部分不符合预期 + 不符合预期 | ≥10% |

- **校准目标**：通过部门级+公司级校准，确保最终分布符合3-6-1规则。
- **系统校验**：校准页面实时显示当前分布占比，A档超过30%或C档不足10%时高亮预警。

#### 3.4.2 流程概览

```
┌─────────┐    ┌─────────┐    ┌─────────────────┐    ┌─────────┐    ┌──────────────────────┐    ┌─────────┐
│ 目标确认 │ → │  自评   │ → │ 互评名单Leader确认 │ → │  互评   │ → │ 上级评估（业绩+价值观） │ → │ 绩效校准 │
└─────────┘    └─────────┘    └─────────────────┘    └─────────┘    └──────────────────────┘    └─────────┘
                                                                                                       ↓
                                                                                            ┌──────────────────────┐
                                                                                            │ 公司级审批（HR→CEO）  │
                                                                                            └──────────────────────┘
                                                                                                       ↓
                                                                                            ┌──────────────────────┐
                                                                                            │      绩效反馈         │
                                                                                            └──────────────────────┘
                                                                                                       ↓
                                                                                            ┌──────────────────────┐
                                                                                            │      结果公布         │
                                                                                            └──────────────────────┘
```

#### 3.4.3 环节1：自评
- **执行人**：被考核员工本人。
- **填写内容**：
  - 重点工作回顾/总结（文本，支持富文本）。
  - **业绩评分**：1-5分，0.25分段，需说明关键成果。
  - **价值观自评**：甲/乙/丙，评"甲"必须提供具体事例。
  - 做得好的部分 / 待改进的部分（需与数据/案例关联，拒绝流水账）。
- **可见性**：提交后，直属上级可见；员工本人可随时修改（在截止时间前）。

#### 3.4.4 环节2：互评名单确认
- **执行人**：直属上级 / 部门Leader。
- **流程**：
  1. 员工在自评时/自评后，选择想要邀请的互评人（不超过5人）。
  2. Leader收到待审核列表，可**新增**或**删除**互评人员。
  3. Leader确认后，系统正式发起互评任务。
- **注意**：互评名单确认在自评环节之后，互评环节之前。

#### 3.4.5 环节3：互评（360°环评）

##### 互评规则

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  员工自选邀请    │ →  │ Leader审核确认   │ →  │   系统发起互评   │
│  （不超过5人）   │     │ （可增删人员）   │     │                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

- **评分维度**：与自评/上级评估一致——**业绩（1-5分，0.25分段）+ 价值观（甲/乙/丙）**。
- **互评信息可见性**：
  - **被评人本人不可见**收到的互评内容（包括评分和评语）。
  - 互评结果仅对 **直属上级、部门Leader、HRBP** 可见，用于辅助上级评估。

##### 匿名主动评价（额外通道）
- **场景**：未被邀请为正式互评人的员工，也可以主动对某位同事发起评价。
- **规则**：
  - 评价视为**匿名评价**。
  - 评价内容仅对 **HRBP 和部门Leader** 可见，**直属上级不可见**。
  - 被评人本人不可见。
- **用途**：作为管理者了解团队氛围和员工协作情况的补充信息，但不纳入正式互评统计。

##### 互评执行
- **评价人操作**：对被评人的业绩（1-5分）和价值观（甲/乙/丙）进行评分，撰写评语。
- **评价人可见性**：评价人仅知道自己评价了谁，不知道其他评价人是谁。
- **手松手紧提示**（V1.1进阶）：系统统计每位评价人的打分分布，对明显偏松或偏紧的评价人给出提示，帮助上级评估时参考。

#### 3.4.6 环节4：上级评估
- **执行人**：被考核人的直属上级。
- **参考信息**：上级评估页面需汇总展示以下信息供参考：
  - 员工基本信息、职级、司龄。
  - 绩效目标内容。
  - 员工自评内容及评分（业绩+价值观）。
  - 收到的正式互评内容及评分（含打分分布）。
  - 匿名评价（仅HRBP和部门Leader可见，上级评估页面不展示）。
  - 员工历史绩效结果（如有）。
- **上级操作**：
  - **业绩评分**：1-5分，0.25分段，需说明关键成果。
  - **价值观评级**：甲/乙/丙，评"甲"必须提供具体事例。
  - 撰写综合评语。
  - 给出初步绩效等级（业绩等级+价值观等级）。
- **提交后**：结果进入「校准」环节，上级本人仍可修改（在校准完成前）。

#### 3.4.7 环节5：绩效校准（Calibration）

##### 校准机制

- **校准层级**：**部门内校准 + 公司级审批**。
- **审批链**：**串行审批**，部门Leader校准 → HR负责人审批 → CEO审批。

```
上级初评（业绩分+价值观等级）
    ↓
部门Leader 校准（部门级）—— 可修改业绩分/价值观等级，需填写原因
    ↓
HR负责人 审批（公司级第一级）—— 可批准或驳回
    ↓
CEO 审批（公司级第二级）—— 可批准或驳回
    ↓
结果锁定
```

- **驳回规则**：HR负责人或CEO驳回时，需填写驳回原因，退回至部门Leader重新校准。

##### 校准视图
- **列表视图**：部门成员名单、上级初评等级、部门校准后等级、审批状态（待HR审批/待CEO审批/已通过/已驳回）。
- **矩阵视图**：横轴-部门/职级，纵轴-绩效等级，直观展示分布情况。
- **分布规则校验（3-6-1）**：
  - **A档（前30%）**：优秀 + 部分超出预期，≤30%。
  - **B档（中间60%）**：符合预期，≈60%。
  - **C档（后10%）**：部分不符合预期 + 不符合预期，≥10%。
  - 系统高亮提示当前分布是否合规，点击异常区域可下钻至人员列表。

##### 校准操作
- **部门级校准人**：可修改员工的业绩分和价值观等级，需填写修改原因（留痕）。
- **公司级审批人（HR负责人、CEO）**：查看部门级校准结果，可批准或驳回。
- 所有修改记录完整留痕（操作人、时间、原值、新值、原因）。
- **校准完成**：CEO审批全部通过后，结果锁定进入「反馈」环节。

#### 3.4.8 环节6：绩效反馈（1v1沟通）
- **执行人**：直属上级。
- **流程**：
  1. 上级发起反馈面谈预约（可选，可线下沟通后线上记录）。
  2. 上级在系统中填写面谈记录：员工优势、待改进项、下一阶段目标/期望。
  3. 员工查看自己的最终绩效结果和面谈记录。
  4. 员工可点击「确认收到」或「有异议」（如有异议流程，可配置）。
- **可见性**：最终绩效结果在反馈环节完成后对员工可见。

### 3.5 消息通知中心

#### 3.5.1 消息类型

| 触发场景 | 接收人 | 消息内容示例 |
|----------|--------|--------------|
| 绩效项目启动 | 全体参与人 | 「2025下半年绩效考核已启动，请在1月13日前完成自评。」 |
| 互评人员待审核 | 直属上级/部门Leader | 「您有3位下属的互评邀请待审核，请尽快确认。」 |
| 环节即将截止（3天/1天） | 未完成人 | 「您的自评任务将于1天后截止，请尽快完成。」 |
| 被分配互评任务 | 评价人 | 「您有3个互评任务待完成，请在1月20日前提交。」 |
| 上级评估待办 | 直属上级 | 「您有5位下属的评估任务待完成，请在1月27日前提交。」 |
| 校准任务提醒 | 校准人 | 「绩效校准已开启，请在2月5日前完成。」 |
| 审批待办 | HR负责人/CEO | 「XX部门的绩效校准结果待您审批。」 |
| 绩效结果公布 | 员工本人 | 「您的2025下半年绩效结果已公布，请查看。」 |
| 催办通知 | 被催办人 | 「HRBP提醒您尽快完成自评任务。」 |
| 结果变更通知 | 原评估人/被评人 | 「您的绩效结果在校准环节被调整，请查看。」 |
| 审批驳回通知 | 被驳回人 | 「您的校准结果被CEO驳回，请查看原因并重新调整。」 |

#### 3.5.2 消息推送方案（已确认）

> **决策结论**：绩效系统作为独立企微自建应用，直接调用企业微信官方 API 发送应用消息，不通过 WorkBuddy 中转。

**理由**：
- 绩效系统本身需注册为企微应用实现 OAuth 免登，已有 `access_token` 和 `agentid`，发消息是顺手能力。
- 直接调企微 API 开发成本极低，无需研究 WorkBuddy 的 Webhook 能力。
- 少一层中间依赖，更稳定可控。
- 员工体验无差别（都是企微应用消息）。

**降级机制**：
- 系统内部维护 `notification_logs` 表，记录所有待发送消息及状态。
- 企微消息发送失败时自动重试3次。
- 连续失败可降级为邮件通知 HRBP 人工跟进。

### 3.6 历史绩效与人才盘点

#### 3.6.1 历史绩效查询
- **个人视角**：员工可查看自己所有历史周期的绩效目标、自评、上级评语、最终等级。
- **管理视角**：上级可查看下属的历史绩效趋势；部门Leader/HRBP可按部门查看历史分布。
- **数据导出**：HRBP可导出指定周期、指定范围的绩效数据（脱敏处理，按权限）。

#### 3.6.2 人才盘点（九宫格）—— 预留架构

> **确认信息**：人才盘点至少一年做一次，具体方案暂不确认，但系统需预留扩展能力。

- **预留维度**：
  - **横轴**：绩效结果（已具备，含业绩等级+价值观等级）。
  - **纵轴**：潜力/价值观/能力（预留字段和接口，V1.2实现）。
- **预留能力**：
  - 员工档案预留「潜力评估」字段。
  - 绩效项目配置预留「是否启用潜力评估」开关。
  - 报表模块预留九宫格可视化组件。
- **V1.2实现**：
  - 九宫格视图：自动将人员落入各象限。
  - 人员标签：明星员工、核心骨干、待提升等（可按公司人才策略自定义）。

### 3.7 企业微信集成

#### 3.7.1 集成方式
- **嵌入企业微信工作台**：以「H5应用」或「小程序」形式嵌入企业微信工作台，员工点击即可进入。
- **免登**：通过企业微信 OAuth2 授权，实现无感登录（无需单独账号密码）。
- **组织架构同步**：通过企业微信 API 同步部门树、人员信息、汇报关系。
- **消息推送**：绩效系统作为独立企微自建应用，直接调用企微「发送应用消息」API。

#### 3.7.2 移动端适配
- 核心功能（填写自评/互评、查看结果、接收通知）必须适配移动端，方便员工随时随地操作。
- 校准、报表分析等管理操作可优先PC端，移动端支持查看。

## 四、非功能需求

### 4.1 性能
- 系统支持100人规模并发使用无压力。
- 页面加载时间 < 2s（常规网络）。

### 4.2 安全
- 所有数据传输使用HTTPS。
- 敏感操作（如校准修改结果）需留痕（操作人、时间、前后值、原因）。
- 数据备份：每日自动备份数据库。
- 离职人员数据保留策略：离职后保留历史绩效数据，但不再可见于当前流程。

### 4.3 可用性
- 系统可用性目标：99.5%（考核高峰期需保障稳定）。
- 考核期间应禁用非紧急的系统升级/维护。

### 4.4 可扩展性
- 架构设计预留扩展空间，未来可支持更多考核模式（如OKR+绩效、KPI等）。
- 评估维度、评分规则、流程环节均支持配置化，不硬编码。
- 人才盘点模块预留潜力评估字段和接口。
