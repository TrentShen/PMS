# 试用期管理模块 —— 产品需求文档

> 状态：V1.0 已实现  
> 适用对象：100 人以内规模企业  
> 最后更新：2026-06-24

---

## 一、背景与目标

### 1.1 背景

公司试用期管理目前主要依赖 Excel 和线下沟通，缺乏系统化的目标制定、进度跟踪和转正评估工具。新员工入职后，试用期目标、评估记录、转正建议分散在不同文档和聊天记录中，难以追溯。

### 1.2 目标

建立一个**独立于常规绩效周期**的轻量级试用期管理模块，实现：

1. 试用期计划根据员工入职信息自动创建
2. 员工在线填写试用期目标，上级审批确认
3. 临转正前，上级基于目标完成试用期评估并给出转正建议
4. 本人、上级、HRBP、部门 Leader、超管按权限查看

---

## 二、用户角色与权限

| 角色 | 权限 |
|---|---|
| **普通员工** | 查看自己的试用期计划；填写/修改自己的目标（审批前） |
| **直属上级** | 查看下属的试用期计划；审批目标；提交试用期评估 |
| **部门 Leader** | 查看本部门及子部门员工的试用期计划（只读） |
| **HRBP** | 查看管辖范围内的试用期计划；可修改计划起止/延期；可代操作 |
| **超级管理员** | 全局可见；可修改所有数据 |

## 三、试用期生命周期

### 3.1 计划创建

- 系统自动根据 `User.hired_at` + `User.probation`（默认 6 个月）创建试用期计划
- 每日凌晨 02:30 定时同步，为 `employee_status='probation'` 且没有计划的员工自动创建
- HR/超管可手动点击"同步试用期计划"触发

### 3.2 目标填写与审批

```
员工填写目标（可保存草稿）
    ↓
员工提交目标
    ↓
状态变为"目标待审批"
    ↓
上级批准 → 状态变为"试用期进行中"
    ↓
上级驳回 → 状态回到"填写目标中"，员工可修改后重新提交
```

### 3.3 试用期评估

- 临转正前（默认 end_date 前 7 天），系统自动将状态标记为"临转正，待评估"
- 上级收到提醒后，进入详情页填写评估意见并选择转正建议
- 转正建议选项：
  - **建议转正**
  - **建议淘汰**
  - **待定/其他**

### 3.4 计划变更

HR/超管可修改试用期计划：
- 延期：修改结束日期，状态变为"已延期"
- 状态调整：用于人工干预流程

## 四、页面原型

### 4.1 试用期管理列表页（`/probation`）

- **可见角色**：HR/超管/Leader/有下属的人员
- **功能**：
  - 列表展示：姓名、部门、直属上级、开始日期、结束日期、剩余天数、状态
  - 搜索：按姓名搜索
  - 筛选：按状态筛选
  - 操作：查看详情
  - HR 额外：手动同步试用期计划按钮

### 4.2 试用期详情页（`/probation/:userId`）

- **基本信息卡片**：姓名、部门、直属上级、试用期起止、剩余天数、状态
- **目标区域**：
  - 员工可编辑/提交目标（审批前）
  - 所有人可查看已提交/已批准的目标
  - 上级可审批目标（批准/驳回）
- **评估区域**：
  - 上级可填写转正建议和评估意见
  - 已完成的评估展示评估人、时间、结果
- **HR 操作区域**：
  - HR 可修改计划状态和结束日期

### 4.3 首页入口（`/`）

- 员工：若自己有试用期计划，首页显示"我的试用期"卡片
- Leader/HR：首页显示"试用期管理"快捷入口

## 六、状态定义

### 6.1 计划状态

| 状态码 | 说明 |
|---|---|
| `draft` | 计划已创建，目标未填写 |
| `objective_draft` | 员工填写目标中，未提交 |
| `objective_pending_review` | 目标已提交，待上级审批 |
| `in_progress` | 目标已批准，试用期进行中 |
| `pending_evaluation` | 临转正前，待上级评估 |
| `completed` | 评估已完成 |
| `extended` | 已延期 |

### 6.2 目标状态

| 状态码 | 说明 |
|---|---|
| `draft` | 草稿 |
| `pending_review` | 待审批 |
| `approved` | 已批准 |
| `locked` | 已锁定 |

### 6.3 转正建议

| 状态码 | 说明 |
|---|---|
| `regular` | 建议转正 |
| `eliminate` | 建议淘汰 |
| `pending_other` | 待定/其他 |

## 九、消息通知

### 9.1 通知原则

- **只做状态驱动的即时提醒**：试用期计划创建、目标提交/审批/驳回、临转正待评估、评估完成等关键节点触发一次企微应用消息。
- **不做周期性入职节点提醒**：入职 5/10/90/140 天等周期性提醒继续由外部 HR-bot 负责，PMS 内不重复推送。
- **不引入群机器人/webhook**：所有通知通过企微自建应用 `agentid` 定向推送给个人。

### 9.2 即时通知场景

| 事件 | 接收人 | 标题 | 说明 |
|---|---|---|---|
| 试用期计划创建 | 员工本人 | 试用期计划已创建 | 系统或 HR 为员工创建计划后触发 |
| 员工提交目标 | 直属上级 | 试用期目标待审批 | 员工点击"提交审批"后触发 |
| 上级批准目标 | 员工本人 | 试用期目标已确认 | 目标进入"试用期进行中"后触发 |
| 上级驳回目标 | 员工本人 | 试用期目标被驳回 | 附驳回原因，员工需修改后重新提交 |
| 临转正待评估 | 直属上级 + HRBP | 试用期评估提醒 | 系统标记为 pending_evaluation 时触发 |
| 上级提交评估 | 员工本人 + HRBP | 试用期评估已完成 | 评估完成并给出转正建议后触发 |

### 9.3 可复用的消息推送服务

- 后端新增 `pms/services/notification.py`，统一封装企微 text / textcard / markdown 三种消息格式。
- 所有发送记录写入 `NotificationLog`（成功/失败均有记录）。
- 未来绩效周期各关键节点（自评、互评、上级评估、校准、审批、反馈、发布）可直接复用该服务。

## 十、与绩效周期的关系

试用期管理**完全独立**于绩效周期：

- 不使用 `PerformanceCycle` 和 `Objective` 表
- 试用期目标不绑定任何绩效周期
- 第二阶段"历史目标导入"才会涉及绩效周期目标

---

## 十一、验证标准

1. HR 或系统为一个试用期员工自动创建计划
2. 员工登录后能看到自己的试用期计划，填写目标并提交
3. 直属上级收到审批待办，批准目标
4. 临转正前，计划状态变为"待评估"，上级提交评估意见和转正建议
5. 员工查看自己的试用期结果
6. HR/Leader 能在列表页查看管辖范围内的试用期状态
